Datenschutzerklaerung (Demo)

Diese Webseite ist ein Lernprojekt und dient ausschliesslich Demonstrationszwecken. Es werden keine Nutzerkonten gefuehrt und keine personenbezogenen Daten aktiv an einen Server uebermittelt. Trotzdem ist es sinnvoll, transparent zu erklaeren, welche Daten technisch im Browser verarbeitet oder gespeichert werden koennen.

Wichtig: Die CO2-Daten auf der Datenseite sind fiktiv. Die Filter- und Sortierfunktionen laufen komplett lokal im Browser, damit die Funktionsweise nachvollziehbar bleibt und keine unnoetige Datenübertragung entsteht.

1. Datenverarbeitung

Beim Aufruf der Webseite werden die Inhalte statisch geladen. Die Eingaben in den Filterfeldern (Land, Unternehmen) werden ausschliesslich im Browser verarbeitet, um die Tabelle zu filtern und die Trefferanzahl anzuzeigen. Es gibt keinen Login, kein Formular-Submit und keine Datenbankanbindung.

Zur Benutzerfreundlichkeit speichert die Anwendung eine kleine Layout-Einstellung, damit sich das Menue nicht bei jedem Seitenwechsel wieder zuruecksetzt.

  • localStorage: Speichert die Menue-Position (links oder rechts), damit sich das Layout nicht bei jedem Seitenwechsel zuruecksetzt.
  • Filtereingaben: Werden nur lokal im Browser genutzt, um die Tabelle zu filtern und zu sortieren. Es findet keine Server-Uebertragung dieser Eingaben statt.

Wenn der Browser-Cache oder localStorage geloescht wird, gehen diese Einstellungen verloren. Das ist gewollt, da die Anwendung keine dauerhafte Datenspeicherung benoetigt.

2. Cookies / Tracking

Diese Demo nutzt keine Tracking-Cookies und kein Analyse-Tracking. Es werden keine Nutzerprofile erstellt und keine Tracking-IDs gesetzt. Die Navigation und Filterfunktionalitaet funktioniert ohne Cookies.

Hinweis: Der Browser kann technisch eigene Cookies aus anderen Webseiten gespeichert haben. Diese stehen nicht im Zusammenhang mit dieser Demo und werden von der Anwendung nicht ausgelesen oder ausgewertet.

3. Drittanbieter (CDN)

Bootstrap wird ueber ein Content Delivery Network (CDN) eingebunden. Das bedeutet: Dein Browser laedt CSS/JS-Dateien direkt vom CDN-Anbieter, um das Layout und die Komponenten bereitzustellen. Dadurch kann der CDN-Anbieter technisch z. B. die IP-Adresse sehen, weil jede Datei-Anfrage über eine Netzwerkverbindung erfolgt.

Alternative: Man kann Bootstrap auch lokal im Projektordner hosten. Das wuerde externe Requests reduzieren, ist aber fuer eine Demo nicht zwingend.

4. Sicherheit

Eingaben werden so verarbeitet, dass kein injizierter Code ausgefuehrt wird. Die Tabelle wird ueber sichere DOM-Methoden aufgebaut (z. B. createElement) und Inhalte werden als Text ausgegeben (textContent statt innerHTML). Damit wird verhindert, dass Nutzereingaben als HTML interpretiert werden.

Zusaetzlich werden Filtereingaben vereinfacht/bereinigt (z. B. durch Längenbegrenzung und Reduktion ungewoehnlicher Zeichen), um unnoetig riskante Zeichenfolgen weiter zu entschärfen. Entscheidend bleibt jedoch die sichere Ausgabe via textContent.
